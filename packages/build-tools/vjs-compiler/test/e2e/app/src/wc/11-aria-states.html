Level 11: ARIA States - Web Component

Tests ARIA attribute selectors for accessibility-driven styling

Test Instructions:

  • Normal state: Button is blue with 90% opacity
  • Hover: Button becomes solid blue (#2563eb) and scales to 110%
  • Disabled state: Click "Set Disabled" - button becomes gray, 50% opacity, cursor not-allowed
  • Disabled hover: When disabled, hover does NOT scale or change color (overridden)
  • Busy state: Click "Set Busy" - button at 70% opacity with pulse animation
  • Pressed state: Click "Set Pressed" - button becomes dark blue (#1e40af) with white ring
  • Multiple states: Try combining states (e.g., busy + pressed)
  • Icon aria-busy: When button is busy, icons should scale down to 90%
  • Console: Should have no errors
